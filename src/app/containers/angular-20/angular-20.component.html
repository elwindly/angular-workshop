Angular 20 new features 2025 MAY 26

  • Stabilized Signal-based APIs: effect(), toSignal(), toObservable(), and linkedSignal() now stable and production-ready
  • Control Flow Syntax: Stabilized @'if', @'for', @'switch', @'case', and @'let' for cleaner templates; *ngIf, *ngFor, *ngSwitch deprecated (removal in v22)
  • Zoneless Change Detection: Moved to developer preview with improved performance without Zone.js 20.2 production ready
  • Resource API (experimental): resource() and httpResource() for signal-based async state and HTTP requests with automatic cancellation and reactive params
  • SSR: Incremental hydration and route-level render mode (ServerRoute, RenderMode.Server/Client/Prerender, getPrerenderParams()) promoted to stable
  • Experimental MCP server to help with LLM codegen
  • Template Expression Improvements: Support for exponentiation operator (**), in keyword, void operator, and tagged template literals in expressions
  • Chrome DevTools: Angular-specific track in the Performance panel (ng.enableProfiling()) for component rendering and change detection visibility
  • Angular DevTools: Route visualization, signal graph, and defer block / incremental hydration inspection
  • Extended diagnostics: Invalid nullish coalescing, missing structural directive imports, and uninvoked track function in @'for'
  • Style guide and CLI: Optional filename and class name suffixes; new projects omit .component, .service, etc. by default
  • Host bindings: Type checking and language service for the host object in component metadata (typeCheckHostBindings in tsconfig; default in v21)
  • GenAI: llms.txt for LLM discovery and angular.dev/ai guides and videos for building GenAI applications
  • CDK Drag&Drop improvement to allow copying items between lists Link to the official page
  • Dynamic Component Features: Add input binding support, two-way bindings, and directives on dynamically-created components
  • Router Enhancements: Asynchronous redirects support, ability to directly abort navigation, improved type safety on route guards
  • Forms Improvements: markAllAsDirty method added, ability to reset forms without emitting events
  • Animations: New animation APIs for more dynamic and flexible animations Link to the official page
  • Experimental Vitest support in Angular CLI (watch mode and browser testing); promoted to stable default in v21
